
# Loop Optimizations

## Why Optimize Loops?

!!! quote "Pareto 原则（二八定律）"
	"大多数程序把 80% 的运行时间花在 20% 的代码上。" —— 而这 **20% 的热点代码，绝大多数是循环**。

因此优化循环往往能带来最大的收益。常见的循环优化技术有：

- **代码外提 / 提升（hoisting / code motion）**：把循环不变的计算移到循环外；
- **强度削减（strength reduction）**：把昂贵运算（乘法）换成廉价运算（加法）；
- **循环展开（loop unrolling）**：把循环体复制多份以摊薄循环控制开销；
- 以及**归纳变量分析与消除、数组边界检查消除**等。

所有这些优化都依赖一个前提：**先在控制流图中准确地"找到"循环**。本章正是围绕"如何识别循环、如何在循环上做优化"展开。

## Loops in a Control-Flow Graph

!!! info "循环的图论定义"
	控制流图中的一个**循环（loop）**是一组节点 $S$，其中包含一个**头节点（header）$h$**，满足：
	
	1. 从 $S$ 中**任意节点**都有一条有向路径到达 $h$；
	2. 从 $h$ 出发有一条有向路径到达 $S$ 中**任意节点**；
	3. 除了到 $h$ 的边以外，**没有**从 $S$ 外部进入 $S$ 的边（即 **单入口**）。

- **循环入口节点（entry node）**：有一个**来自循环外**的前驱；
- **循环出口节点（exit node）**：有一个**位于循环外**的后继。

!!! note "一个循环**只有一个入口**（即头节点 h），但**可以有多个出口**。"

## Dominators

要优化循环，必须先在流图里找到它们；**支配（dominator）**概念正是为此服务。

!!! info "支配（Dominator）"
	节点 $d$ **支配** 节点 $n$，当且仅当从起始节点 $s_0$ 到 $n$ 的**每一条**有向路径都**必须经过** $d$。（$s_0$ 是控制流图的起始节点。）
	
	- **每个节点都支配它自己**；
	- 一个节点 $n$ 可以有**多个**支配节点。

**求支配集的算法**：设 $D[n]$ 为支配 $n$ 的节点集合，则

$$
D[s_0] = \{s_0\}, \qquad D[n] = \{n\} \cup \left( \bigcap_{p \in pred[n]} D[p] \right) \quad (n \neq s_0)
$$

用**迭代法**求解：把 $D[n]$（$n \neq s_0$）**初始化为图中所有节点**，反复套用上式直到不动点。（直观理解：$n$ 的支配者，是"$n$ 自己" + "$n$ 的所有前驱的支配者的交集"。初始取全集、单调收缩，保证收敛到最大不动点。）

### 直接支配者（Immediate Dominator）

!!! info "定理"
	在一个连通图中，若 $d$ 支配 $n$，且 $e$ 也支配 $n$，那么必有 **$d$ 支配 $e$** 或 **$e$ 支配 $d$**（$n$ 的所有支配者排成一条链）。

由该定理，除 $s_0$ 外每个节点 $n$ 都有**唯一**的**直接支配者 $idom(n)$**，满足：

- $idom(n) \neq n$；
- $idom(n)$ 支配 $n$；
- $idom(n)$ **不支配** $n$ 的任何**其它**支配者（即它是离 $n$ "最近"的支配者）。

### 支配树（Dominator Tree）

为流图的每个节点画一个节点，并为每个 $n$ 画一条从 $idom(n)$ 到 $n$ 的边——得到的图**一定是一棵树**。

![[cp_topic18_2.png]]

（**为什么是树？** 因为每个节点恰有一个直接支配者，即恰有一个父节点，且 $s_0$ 为根、无环——这正是树的定义。）左为流图 (a)，右为其支配树 (b)。

## Natural Loops

我们特别关心循环是否**只有单一入口**：

!!! note "单入口的好处"
	单入口 ⇒ 每次进入循环迭代时，开头都满足**某些确定的初始条件** ⇒ **便于优化**。这一诉求催生了**自然循环（natural loop）**的定义。

!!! info "回边与自然循环"
	- **回边（back edge）**：一条流图边 $n \to h$，其中 **$h$ 支配 $n$**，称为回边。
	- 回边 $n \to h$ 的**自然循环**：所有满足"$h$ 支配 $x$，且存在一条从 $x$ 到 $n$、**不经过 $h$** 的路径"的节点 $x$ 的集合。该循环的**头节点就是 $h$**。

![[cp_topic18_1.png]]

以上图为例：

- 回边 $10 \to 5$ 的自然循环包含节点 **5、8、9、10**，其中 **{8, 9}** 是嵌套在内的循环；
- 一个头节点 $h$ 可以是**多个**自然循环的头：$3 \to 2 \Rightarrow \{2,3\}$，$4 \to 2 \Rightarrow \{2,4\}$；
- **$\{2,3,4\}$ 构成一个循环，但它不是自然循环**（它有 3 和 4 两个"入口方向"，不是单入口）。

!!! note "嵌套循环（Nested Loops）"
	若 A、B 是两个循环，头节点分别为 $a$、$b$，且 $a \neq b$、$b$ 在 A 中，则 B 的节点是 A 的节点的**真子集**。称 **B 嵌套在 A 内**，B 是**内层循环（inner loop）**。

### 循环嵌套树（Loop-Nest Tree）

可以为程序中的所有循环构造一棵**循环嵌套树**：

1. 计算流图的支配关系（dominators）；
2. 构造支配树；
3. 找出所有自然循环，从而得到所有循环头节点；
4. 对每个循环头 $h$，把 $h$ 的所有自然循环**合并**成单个循环 $loop[h]$；
5. 构造循环头的树：若 $h_2$ 在 $loop[h_1]$ 中，则 $h_1$ 在树中位于 $h_2$ 之上。

![[cp_topic18_3.png]]

!!! quote "可以认为**整个过程体**是一个位于循环嵌套树**根部**的「伪循环（pseudo-loop）」。"

### 循环 Preheader（前置头）

许多循环优化需要在**循环执行之前**插入语句（例如循环不变式外提 hoisting，就是把语句从循环内移到循环正前方）。为了有一个统一的"插入位置"：

!!! info "插入 Preheader"
	新建一个初始为空的 **preheader 节点 $p$**，把所有**来自循环外部**的边 $y \to h$ 都**改为指向 $p$**（循环内部到 $h$ 的回边保持不变）。

![[cp_topic18_4.png]]

左图 (a) 是原图，右图 (b) 插入了 preheader $p$：外部的 2、3 先进入 $p$，再由 $p$ 进入头节点 4。

## Loop-Invariant Computations

如果循环中有语句 $t \leftarrow a \oplus b$，其中 $a$ 每次循环取值相同、$b$ 每次也相同，那么 $t$ 每次也取相同的值——我们希望把这个计算**外提**到循环外。问题是：**怎么知道 $a$ 每次取值都相同？**

!!! info "循环不变式（Loop-Invariant）的定义"
	定义 $d:\ t \leftarrow a_1 \oplus a_2$ 在循环 $L$ 中是**循环不变**的，当且仅当对**每个**操作数 $a_i$，下列之一成立：

	- $a_i$ 是**常量**；
	- **或** 所有能到达 $d$ 的 $a_i$ 的定义都在循环**外部**；
	- **或** 只有一个 $a_i$ 的定义能到达 $d$，且那个定义本身是**循环不变**的。

最后一条是递归的，因此要用**迭代算法**反复扫描循环，直到不再有新的不变式被识别（不动点）。

## Hoisting

假设 $t \leftarrow a \oplus b$ 是循环不变的，**能否**把它外提（hoist）到 preheader？**不总是能！** 下面四个例子里，只有 (a) 可以安全外提：

![[cp_topic18_5.png]]

!!! warning "外提 $d:\ t \leftarrow a \oplus b$ 到 preheader 末尾的判据（三条须**同时**满足）"
	1. **$d$ 支配所有"$t$ 在其出口 live-out"的循环出口**；
	2. **且** 循环中**只有一个** $t$ 的定义；
	3. **且** $t **不是** preheader 的 live-out（即外提前 $t$ 在 preheader 处不活跃）。

对照上图：

- **(b) 不能**——$d$ 不支配所有循环出口（`if i≥N goto L2` 使 def 不一定每次都执行），外提会**改变语义**（本来可能不执行的计算被强制执行了）；
- **(c) 不能**——循环中有**不止一个** $t$ 的定义；
- **(d) 不能**——在 def 之前就有对 $t$ 的**使用**（$t$ 在 preheader live-out），外提会破坏那次使用读到的值。

!!! note "两个补充注意点"
	- **隐式副作用**：若 $a \oplus b$ 可能**抛出算术异常**或有其它副作用，上述规则需要修改（不能贸然把可能抛异常的计算提前）。
	- **while 循环 → repeat-until**："$d$ 支配所有 live-out 出口"这一条，会**阻止很多 while 循环中的计算被外提**（while 循环里往往**没有任何语句支配循环出口节点**）。先把 while 改写成 repeat-until，再做外提会有效得多。

## Induction Variables

有些循环里存在：

- 一个被**递增或递减**的变量 $i$；
- 一个 $j = i \cdot c + d$，其中 $c$、$d$ 循环不变。

那么就能**不引用 $i$** 而直接维护 $j$：每当 $i$ 增加 $a$，就让 $j$ 增加 $a \cdot c$。

!!! info "基本归纳变量与派生归纳变量"
	- **基本归纳变量（basic induction variable）$i$**：在循环 $L$（头节点 $h$）中，$i$ 的**唯一**定义形如 $i \leftarrow i + c$ 或 $i \leftarrow i - c$，其中 $c$ 循环不变。
	- **派生归纳变量（derived induction variable）$k$**：用**三元组 $(i, a, b)$** 表示，含义是 $k = a + i \cdot b$（$i$ 为基本归纳变量，$a$、$b$ 循环不变）。

例如 $j \leftarrow i \cdot 4$ 对应三元组 $(i, 0, 4)$；其后 $k \leftarrow j + a$（$a$ 循环不变）则 $k = i \cdot 4 + a = a + i\cdot 4$，对应 $(i, a, 4)$。

!!! note "线性归纳变量（Linear Induction Variable）"
	一个归纳变量若在循环的**每一次迭代**中都改变**相同的**（常量或循环不变的）量，就是**线性**的。如果基本归纳变量 $i$ 在不同迭代被加上**不同**的量，它就**不是线性**的——此时派生变量 $j$ 有时 $=i\cdot 4$、有时会被暂时"落下"，无法简单维护。

### 归纳变量的检测（Detection）

!!! info "检测规则"
	- **基本归纳变量**：$i$ 在循环 $L$（头 $h$）中是基本归纳变量，当且仅当 $i$ 在 $L$ 内的唯一定义形如 $i \leftarrow i \pm c$（$c$ 循环不变）。
	- **派生归纳变量**：$k$ 是派生归纳变量，当且仅当
		1. $k$ 在 $L$ 内**只有一个定义**，形如 $k \leftarrow j \cdot c$ 或 $k \leftarrow j + d$（$j$ 是归纳变量，$c, d$ 循环不变）；
		2. 若 $j$ 是 $i$ 族（family）中的派生归纳变量，则还需：**2.1** 到达 $k$ 的 $j$ 的定义唯一，就是循环里那个；**2.2** 在 $j$ 的定义与 $k$ 的定义之间的**任何路径**上都**没有** $i$ 的定义。

### 强度削减（Strength Reduction）

**乘法比加法昂贵**。我们想把每个"定义形如 $j \leftarrow i \cdot c$"的派生归纳变量换成加法。

!!! info "强度削减步骤"
	对每个三元组为 $(i, a, b)$ 的派生归纳变量 $j$（即 $j \leftarrow a + i \cdot b$）：

	1. 新建变量 $j'$；
	2. 在每个 $i \leftarrow i + c$ 之后，插入 $j' \leftarrow j' + c \cdot b$；其中 $c \cdot b$ 是循环不变表达式，可在 preheader 里预先算好（若 $c$、$b$ 都是常量，乘法在**编译期**完成）；
	3. 把对 $j$ 的（唯一）赋值替换为 $j \leftarrow j'$；
	4. 在 preheader 末尾用 $j' \leftarrow a + i \cdot b$ 初始化 $j'$。
	最后再做**死代码消除（dead-code elimination）**。

**例**：原始循环（求 `s = Σ M[a + i·4]`）：

```text
    s ← 0
    i ← 0
L1: if i >= n goto L2
    j ← i · 4
    k ← j + a
    x ← M[k]
    s ← s + x
    i ← i + 1
    goto L1
L2:
```

对 $j$（三元组 $(i,0,4)$）做强度削减：新建 $j'$、在 `i←i+1` 后加 `j'←j'+4`、把 `j←i·4` 换成 `j←j'`、并在前面初始化 `j'←0`：

```text
    s ← 0
    i ← 0
    j' ← 0
L1: if i >= n goto L2
    j ← j'
    k ← j + a
    x ← M[k]
    s ← s + x
    i ← i + 1
    j' ← j' + 4
    goto L1
L2:
```

再对 $k$（三元组 $(i, a, 4)$）做同样的强度削减（$k' \leftarrow a$ 初始化、`k←k'`、`k'←k'+4`）：

```text
    s ← 0
    i ← 0
    j' ← 0
    k' ← a
L1: if i >= n goto L2
    j ← j'
    k ← k'
    x ← M[k]
    s ← s + x
    i ← i + 1
    j' ← j' + 4
    k' ← k' + 4
    goto L1
L2:
```

### 归纳变量消除（Elimination）

强度削减之后，有些归纳变量**完全不再被使用**，另一些**只在与循环不变量的比较中使用**——这些都可以删除。

!!! info "无用变量（useless variable）"
	变量在循环 $L$ 中是**无用的**，当且仅当：

	- 它在 $L$ 的**所有出口处都是 dead**（死的）；
	- **且** 它的**唯一用途**是在对它自己的定义中（如 `j' ← j' + 4`）。

	**无用变量的所有定义都可以删除。** 在上例中，`j`、`j'` 在强度削减后变得无用（`j` 不再被用，`j'` 只用于自增），可删去。

### 改写比较（Rewriting Comparisons）

!!! info "几乎无用的变量（almost-useless variable）"
	变量 $k$ 是**几乎无用的**，当且仅当：它**只**用于"与循环不变量比较"以及"对自己的定义"中，**且** 同族中存在另一个**非无用**的归纳变量 $c$。
	通过**改写比较**，让它使用那个相关的归纳变量，就能把"几乎无用"变成"无用"，进而删除。

**例**：经强度削减后剩下基本归纳变量 `i`（几乎无用，只用于 `if i≥n`）与相关变量 `k'`：

```text
    s ← 0
    i ← 0
    k' ← a
L1: if i >= n goto L2
    k ← k'
    x ← M[k]
    s ← s + x
    i ← i + 1
    k' ← k' + 4
    goto L1
L2:
```

把比较 `i ≥ n` 改写为 `k' ≥ a + 4·n`（其中 `a + 4·n` 循环不变，外提为 `b ← n·4; c ← a + b`），于是 `i` 变为无用、删除；再做**复制传播（copy propagation）**把 `k ← k'; x ← M[k]` 合并为 `x ← M[k']`：

```text
    s ← 0
    k' ← a
    b ← n · 4
    c ← a + b
L1: if k' >= c goto L2
    x ← M[k']
    s ← s + x
    k' ← k' + 4
    goto L1
L2:
```

整个"归纳变量分析 → 强度削减 → 归纳变量消除 → 改写比较 / 复制传播"流水线的前后对比：

## Array-Bounds Checks

有些语言会在**每次下标操作**自动插入**数组边界检查（array-bounds check）**。我们希望编译器删除那些**可证明冗余**的检查。

!!! warning "删除「所有」冗余边界检查是**不可计算**的。"
	但**很多**数组下标形如 `a[i]`，其中 `i` 是归纳变量——这类情形编译器通常能**理解得足够好**从而优化掉检查。

- 数组的边界一般形如 $0 \le i \;\wedge\; i < N$；
- 从循环 $L$ 中消除一个边界检查的**判据相当复杂**（具体见教材）。其核心思想是：若能静态证明归纳变量 $i$ 在整个循环中始终落在 $[0, N)$ 内，则循环内对 `a[i]` 的检查全部冗余，可提到循环外只做一次（或完全消除）。

## Loop Unrolling

有些循环体**非常小**，以至于大部分时间都花在**递增循环计数器**和**测试退出条件**上。**循环展开（unrolling）**就是把循环体复制两份或多份排在一起，以摊薄这部分控制开销。

!!! info "展开的图变换"
	给定头节点为 $h$、回边为 $s_i \to h$ 的循环 $L$：

	1. 复制节点，造出一个头为 $h'$、回边为 $s_i' \to h'$ 的循环 $L'$；
	2. 把 $L$ 中所有回边从 $s_i \to h$ 改成 $s_i \to h'$；
	3. 把 $L'$ 中所有回边从 $s_i' \to h'$ 改成 $s_i' \to h$。

**朴素展开（slide 42）**：直接复制一份循环体，每一份"原始"迭代仍然各自带着一次自增和一次条件跳转——**等于什么也没省到**：

```text
L1: x ← M[i]
    s ← s + x
    i ← i + 4
    if i < n goto L1' else L2
L1': x ← M[i]
    s ← s + x
    i ← i + 4
    if i < n goto L1 else L2
L2:
```

!!! warning "需要一个「好」的归纳变量：每个增量 $i \leftarrow i + \Delta$ 都**支配循环的每条回边**。"
	这样才能把多份的增量与退出测试**合并（agglomerate）**。

**合并增量与测试（slide 43）**：把两份的 `i←i+4` 合成一次 `i←i+8`、只留一个测试。但这样**很脆弱（fragile）**——**只能处理迭代次数为偶数**的情形：

```text
L1: x ← M[i]
    s ← s + x
    x ← M[i+4]
    s ← s + x
    i ← i + 8
    if i < n goto L1 else L2
L2:
```

**健壮版本（slide 44）**：在前面加一个守卫，主体每次处理 2 个元素（`i←i+8`），最后用一个**尾循环（epilogue）**处理剩下的"奇数"次迭代（`i←i+4`），从而**适用于任意迭代次数**：

```text
    if i < n - 8 goto L1 else L2
L1: x ← M[i]
    s ← s + x
    x ← M[i+4]
    s ← s + x
    i ← i + 8
    if i < n goto L1 else L2
L2: x ← M[i]          // epilogue：处理剩余迭代
    s ← s + x
    i ← i + 4
    if i < n goto L2 else L3
L3:
```

!!! note "朴素展开无收益；合并增量/测试 ⇒ 脆弱（仅限偶数次）；加尾循环执行「奇数」次迭代 ⇒ 健壮。"
